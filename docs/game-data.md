# Game Data Reference

本文档记录当前代码里的植物、元素、怪物基础数据。数值平衡时优先同步这里，避免靠记忆翻代码。

## 通用公式

| 项目 | 规则 |
| --- | --- |
| 植物伤害升级 | 每级 +5%，即 `baseDamage * (1 + 0.05 * (level - 1))` |
| 植物射程升级 | 每级 +1%，即 `baseRange * (1 + 0.01 * (level - 1))` |
| 植物攻速升级 | 每级 +2%，攻速为 0 的植物仍为 0 |
| 灰烬/一次性植物升级 | 默认只看伤害升级；不支持元素升级；范围、延迟、控制时间不随等级提升 |
| 植物/元素升级碎片 | `5 + 5 * (currentLevel - 1)` |
| 植物/元素升级金币 | `100 * currentLevel` |
| 怪物生命等级倍率 | `baseHp * (1 + 0.1 * (monsterLevel - 1))`，所以 level 1 等于基础数据 |
| 怪物护甲等级倍率 | `baseArmorHp * (1 + 0.1 * (monsterLevel - 1))`，仅拥有护甲的怪物生效 |
| 怪物速度 | 当前使用基础速度，不随等级自动提升 |
| 波次间隔 | 默认每波结束后 2 秒开始下一波 |
| 同波怪组时序 | 怪组 `startDelay` 留空时按顺序刷，同组间默认间隔 2 秒；填写 `startDelay` 后按本波开始后的秒数调度，同一秒数会同时刷；同一波只填部分怪组时，未填的怪组继续按前面怪组结束后的顺序自动排 |
| Boss 体型 | 只有怪组勾选 `isBoss` 时才按 `10 + totalMaxHp / 15` 放大；普通怪不随血量变大 |
| Boss 击杀奖励 | 固定 500 阳光，不读取波次配置 |
| 真实护甲 | 未破甲时优先扣护甲，护甲扣完后溢出伤害扣本体；破甲状态下伤害绕过护甲直接扣本体 |
| 无护甲破甲 | 没有护甲条的怪物处于金元素破甲状态时，会受到额外伤害倍率 |
| 冻结与减速 | 冻结单独记录；冻结期间速度为 0，冻结结束后仍可继续保留普通减速 |

## 元素单独释放

单独释放规则：选择元素后点空地，会消耗阳光并进入独立冷却；释放点出现引导图标，2 秒后触发。

| ID | 名称 | 冷却 | 单独释放效果 |
| --- | --- | ---: | --- |
| fire | 火元素 | 30 秒 | 释放点半径 4.4 内造成 `800 + 50 * elementLevel` 伤害 |
| ice | 冰元素 | 20 秒 | 全场造成 `20 + 4 * elementLevel` 伤害，并冻结 4 秒 |
| wind | 风元素 | 20 秒 | 全场造成 `20 + 4 * elementLevel` 伤害，并沿路径击退 1.6 格 |
| gold | 金元素 | 20 秒 | 全场施加破甲，持续 `5 + 0.2 * elementLevel` 秒；无护甲目标受到 `1.5 + 0.1 * (elementLevel - 1)` 倍伤害 |
| electric | 电元素 | 30 秒 | 全场造成 `200 + 20 * elementLevel` 伤害 |
| light | 光元素 | 30 秒 | 立刻获得 400 阳光 |
